About

Travis Cook is a Creative Designer for USAA based in San Antonio, Texas. He received a B.S. in Computer Science from Abilene Christian University in December 2017. As a designer, Travis focuses on Human Centered Design methods, designing for accessiblity, and Human-Computer Interaction. For design work, Travis primarily uses Sketch, Invision, and Adobe CC as a basis before completeing designs in JavaScript, React, HTML, and CSS. Travis often codes in C++, Python, and other languages for programming work.

Coloring Website

Project Requirements - Your coloring website mock-up must:

  • Be a desktop web app
  • Be simple enough for children to use
  • Be friendly and pleasant enough that children will want to use it
  • Have at least two activities (this includes traditional coloring)

Notes - This rapid mock-up presented a new aspect for me: designing for children. When planning my mock-up, I found it interesting to try and envision how a child would respond to the design choices; this thought process led to the use of simple language, buttons with mostly pictures, and a strict avoidance of feature creep. Pharmacy Mobile App

Project Requirements - Your pharmacy app mock-up must:

  • Be a mobile app
  • Allow the user to refill medication
  • Alert users when their medication is ready
  • Allow users to see what drugs they have currently prescribed by their physician
  • Allow users to see see what drugs have been previously prescribed (and when they took them and for how long).
  • One other feature you think would be useful (examples: see related drugs, see generic equivalent of brand name drug that user is taking, chat with a pharmacist, find nearest store with a maps feature, etc.)

Notes - For this rapid mock-up, I decided against using a hamburger menu and instead used icons at the bottom. I chose to do this to increase ease of use and learnability for the app. Instead of hiding options for the user, I wanted them to be clearly visible and present at all times.

Smart Watch Fitness App

Project Requirements - Your fitness app mock-up must:

  • Be an app on a high-end smart watch (Eg. Apple Watch, Moto 360, etc)
  • Allow the user to start and stop a workout
  • Allow the user to view how many calories they've burned, distance travelled, average heart rate so far during the current workout.
  • Allow the user to look through previous workouts and view the stats (listed above: calories burned, distance travelled, average heart rate) from each one
  • One additional feature

Notes - This first major challenge with this project was my lack of familairity with smart watches. To overcome this, I researched how other companies have designed fitness apps for the smart watch. Another major challege was the lack of screen space. This led me to carefully decide which functions should be present on the screen and focus on signifying affordances with icons rather than text. Fitness App

Project Requirements - Your fitness app mock-up must:

  • Be an app on a smart phone (Android, iOS, Windows).
  • Be integrated with the smart watch app.
  • Allow the user to input new workouts manually (e.g. they didn't have their smart watch with them)
  • Allow the user to look through previous workouts and view the stats (listed above: calories burned, distance travelled, average heart rate) from each one
  • One additional feature

Notes - This app was designed to be a companion app for the smart watch app. I kept the same color and signifier theme to create a sense of continuity and reduce the user's gulf of evaluation when using the app. I worked to use icons rather than text for most data input to allow users to scan quickly rather than read. I was tempted to add more options and features such as creating goals and workout templates, but I decided against this as it could lead to feature creep and create a cluttered feel. The full mock-up can be seen at the link below.
